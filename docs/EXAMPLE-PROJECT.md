## Workflow Example

1. **Create/Edit Content**
   ```bash
   # Edit a page
   vim content/pages/services.txt
   ```

2. **Test on Staging**
   ```bash
   git add content/pages/services.txt
   git commit -m "feat(content): Add services page"
   git push origin staging
   ```

3. **Deploy to Production**
   ```bash
   # After testing on staging
   git checkout main
   git merge staging
   git push origin main
   ```

4. **GitHub Actions Automatically**:
   - Detects new `content/pages/services.txt`
   - Creates `meta/pages/services.json` with WordPress ID
   - Deploys content to WordPress via REST API
   - Updates theme settings from `design-tokens.json`

## Important Notes

- **Never commit meta files** - They're auto-generated
- **Always test on staging** before merging to main
- **Use proper Enfold shortcodes** in content files
- **Design tokens** map to Enfold theme options
- **Page slugs** are derived from filenames (e.g., `about.txt` â†’ `/about/`)
